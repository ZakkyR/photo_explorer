# Photo Explorer 設計ドキュメント

## 概要

Windows 向けの画像整理アプリ。エクスプローラーよりも快適に画像を閲覧・タグ付け・選択できる。
Affinity など他の Windows アプリへのドラッグアンドドロップによる画像受け渡しが主要ユースケース。

---

## 技術スタック

| 要素 | 選択 | 理由 |
|---|---|---|
| UI フレームワーク | WPF (.NET 8) | D&D の安定性、画像表示の実績、FileSystemWatcher との親和性 |
| DB | SQLite + EF Core | ローカル保存、軽量、マイグレーション管理 |
| EXIF 操作 | MetadataExtractor + ImageSharp | JPEG/PNG への EXIF タグ書込 |
| 画像読込 | WPF BitmapImage（WIC 経由） | RAW はメーカーコーデックまたは Microsoft Raw Image Extension で対応 |

## 機能詳細

### フォルダ管理
- 「+ フォルダ」ボタンからフォルダ選択ダイアログを開く
- 登録フォルダは SQLite に永続化
- `FileSystemWatcher` でファイル追加・削除を監視し、グリッドをリアルタイム更新
- サイドバーのフォルダを右クリック → 「登録解除」で削除可能

### 画像グリッド
- `WrapPanel` + `VirtualizingStackPanel` で効率的なサムネイル表示
- サムネイルサイズをスライダーで動的変更（小・中・大 の 3 段階程度）
- タグ絞込はメモリ上でリアルタイムフィルタリング（200枚以下のため DB クエリ不要）
- 複数タグを選択した場合は **OR 条件**（いずれかのタグを持つ画像を表示）
- 複数選択対応（Ctrl+クリック、Shift+クリック）

### ドラッグアンドドロップ
- `MouseMove` イベントで `DragDrop.DoDragDrop` を起動
- `DataObject` に `FileDrop` 形式でファイルパスを設定
- Affinity Publisher / エクスプローラー等がファイルとして受け取る
- 複数選択時は選択ファイルすべてをドロップ

### タグ付け
- グリッドの画像を右クリック → コンテキストメニューからタグを追加・削除
- プレビューウィンドウ下部からも操作可能
- JPEG / PNG: `ImageSharp` で EXIF（XMP Keywords）に書き込み
- RAW / その他: SQLite の `ImageTags` テーブルに保存
- EXIF 書込失敗時（読み取り専用等）は自動的に SQLite にフォールバック

### プレビューウィンドウ
- ダブルクリックで独立ウィンドウとして開く（メインウィンドウと独立して操作可能）
- `←` `→` キーで前後の画像に切替
- `ESC` で閉じる
- ウィンドウ位置・サイズを次回起動時に復元
- RAW はメーカーコーデックまたは Microsoft Raw Image Extension（ストア）で表示

### アルバム
- 複数フォルダを仮想的にまとめた論理グループ
- 「+ アルバム」でアルバム名を入力し、既存の登録フォルダから追加するフォルダを選択
- フォルダのときと同じ操作感（タグ絞込・D&D・プレビューすべて同一）
- アルバムから操作してもフォルダ側のデータに反映される

## データフロー

### フォルダ追加
1. ユーザーが「+ フォルダ」→ フォルダ選択ダイアログ
2. `FolderService` が SQLite に登録
3. フォルダ内の全画像をスキャン → `ImageItem` リスト生成
4. `FileSystemWatcher` を登録 → 以降の変更を自動検知

### タグ付け（JPEG/PNG）
1. ユーザーがタグを追加/削除
2. `TagService` が `ImageSharp` で EXIF Keywords を更新
3. 失敗時は SQLite `ImageTags` に保存してユーザーに通知なし（透過的フォールバック）

### D&D
1. グリッドで `MouseMove`（一定距離以上）→ D&D 開始
2. `DataObject(DataFormats.FileDrop, string[])` を生成
3. `DragDrop.DoDragDrop` 呼び出し → OS が他アプリへ転送

## エラー処理

| ケース | 対処 |
|---|---|
| 壊れた/読み込めない画像 | サムネイル枠にエラーアイコンを表示、スキップして続行 |
| EXIF 書込失敗（読み取り専用等） | SQLite にフォールバック、ユーザーへの通知なし |
| 登録フォルダが削除された | サイドバーに警告アイコン表示、グリッドは空に |
| RAW コーデック未インストール | グレー枠＋「コーデック未対応」テキスト表示 |

---

## 対応画像フォーマット

| フォーマット | サムネイル | プレビュー | タグ |
|---|---|---|---|
| JPEG | ○ | ○ | EXIF |
| PNG | ○ | ○ | EXIF |
| WebP | ○ | ○ | SQLite |
| BMP / TIFF | ○ | ○ | SQLite |
| RAW（CR2, NEF, ARW 等） | コーデック次第 | コーデック次第 | SQLite |

---

## 非機能要件

- 起動時間: 3 秒以内（フォルダ 3 件・各 200 枚想定）
- サムネイル生成: バックグラウンドスレッドで非同期実行、UI をブロックしない
- ウィンドウ状態（サイズ・位置・最後に選択したフォルダ）を次回起動時に復元（`ApplicationSettings` / user.config に保存）
